● Ready Queue
Separate FIFO queue per seat category. First-in, first-out allocation.
● Cleaning Queue
Tables enter after payment. Dequeued when staff marks them cleaned.
● Priority Queue
Sorted by priority key. VIP → large groups → small groups.
Algorithm Reference
Ready Queue — FIFO
Each seat category (2, 4, 6, 8, 10) has its own FIFO queue stored in MongoDB, sorted by enqueued timestamp. On customer arrival, the front of the matching queue is dequeued and the table allocated. Time complexity: O(1) enqueue and dequeue.
Cleaning Queue — FIFO
Same structure as the ready queue — one FIFO queue per seat category. Tables enter after payment. On marking cleaned, the system checks the priority queue first; if a matching customer is waiting, the table is allocated directly. Otherwise it re-enters the ready queue.
Priority Queue — Max-Heap
Customers who cannot be immediately seated enter this queue. Priority key: VIP=0, 10-seat=1, 8-seat=2, 6-seat=3, 4-seat=4, 2-seat=5. Ties broken by enqueued timestamp (FIFO within same priority).
